Autoresearch Skill Package

Initializer skill for task-focused autonomous ML research repos

This repository packages a Codex skill named autoresearch. The skill scaffolds a clean, git-initialized autoresearch repository, helps define task packs, and leaves experiment execution to the generated repository.

Mental Model Package Layout Install Scaffold Task Packs Generated Repo Adapters Validation

Mental Model

The skill is an initializer, not the runtime orchestrator. It creates a fresh repository and guides the user through a decision-complete task definition. After scaffolding, the generated repository becomes the runtime authority.

Skill responsibility

Scaffold a repo, initialize git, optionally include an adapter or CSP example, and guide task-pack creation.

Generated repo responsibility

Own campaign scripts, candidate.py contracts, task packs, logs, branches, artifacts, and run-loop instructions.

Task-pack responsibility

Encode the domain-specific objective, data setup, metric, evaluator, constraints, prior methods, and smoke checks.

Current scope: initialization and task-pack definition. Running campaigns, reflection, and reporting are intentionally handled inside the generated repo.

Package Layout

The repository root is only the skill distribution source. The bundled repo template is the single source of truth for generated repos.

Path Purpose
skills/autoresearch/SKILL.md Skill metadata and concise routing instructions.

Install Or Register The Skill

For local development, symlink the skill into the Codex skills directory so edits in this repo are immediately visible. 

mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
ln -s "$(pwd)/skills/autoresearch" "${CODEX_HOME:-$HOME/.codex}/skills/autoresearch"

For a copied install, copy the skill directory instead. 

mkdir -p "${CODEX_HOME:-$HOME/.codex}/skills"
cp -R skills/autoresearch "${CODEX_HOME:-$HOME/.codex}/skills/autoresearch"

Once registered, invoke the skill with $autoresearch or by asking Codex to initialize, scaffold, or define a task-focused autoresearch repository.

Scaffold A Repository

The scaffolder creates a generated autoresearch repo from the bundled template. By default it initializes git on agent/root and creates the initial commit Create a bootstrap-ready autoresearch harness. 

python skills/autoresearch/scripts/scaffold_repo.py \
  --target /path/to/new-repo \
  --name "Project Name"

Optional Claude adapter and CSP example

python skills/autoresearch/scripts/scaffold_repo.py \
  --target /path/to/new-repo \
  --name "Project Name" \
  --adapter claude \
  --include-csp-example
Option Behavior
--adapter claude Copies the Claude Code prompts, settings, and guard hook into .claude/.
--include-csp-example Copies the runnable CSP task pack into examples/csp/.
--no-git Skips git initialization when another system will own the repository.
--force Allows copying into a target with user files after the caller accepts overwrite risk.

Targets that contain only .omx/ or .DS_Store are treated as safe runtime-state directories and are preserved without requiring --force.

Task Packs

A task pack is the complete domain contract for a generated repo. The skill should interview until these decisions are explicit: objective, data, metric, evaluator behavior, allowed artifacts, compute limits, candidate data access, prior methods, novelty boundaries, and smoke checks.

File Role
task.json Machine-readable manifest consumed by scripts.
task.md Problem statement, objective, assumptions, and success criteria.
contract.md Hard rules for data access, outputs, evaluation, and resources.
methods.md Prior methods, measured baselines, and novelty boundary for idea generation.
prepare.py Idempotent setup into data/<slug>/.
evaluate.py Fixed evaluator for candidate artifacts.
requirements.txt Task-specific Python dependencies.
smoke.md Cheap checks before full candidate runs.

Manifest baseline

{
  "slug": "task_slug",
  "name": "Task name",
  "candidate_entry": "candidate.py",
  "metric": {
    "name": "Primary metric",
    "direction": "maximize"
  },
  "defaults": {
    "max_minutes": 120,
    "smoke_seconds": 120,
    "noise_sigma": 0.0
  },
  "artifacts": {
    "data_dir": "data/<slug>",
    "run_subdir": "runs/<run_name>"
  }
}
Do not leave placeholders such as TODO, TBD, example_slug, or vague metric names in generated task packs. Unknown facts should become concrete conservative assumptions.
Before bootstrap, run the task preparation path and score at least one cheap valid baseline through the fixed evaluator. Record the measured command, artifact path, and primary metric in task.md or methods.md.

Generated Repository Contract

The generated repo uses a fixed candidate interface so the orchestrator can compare search nodes consistently across tasks.

Contract item Value
Implementation file candidate.py
Primary score line primary_metric: <float>
Task pack path tasks/<slug>/ or examples/<slug>/
Campaign branches agent/<task_slug>/<run_tag>/*
Artifacts runs/<task_slug>/<run_tag>/<sha>/
Event log runs/<task_slug>/<run_tag>/campaign_events.jsonl
Interactive report runs/<task_slug>/<run_tag>/report.html

Candidate CLI

python candidate.py \
  --run_name staging \
  --max_minutes 120 \
  --task_dir tasks/<slug> \
  --data_dir data/<slug>

Candidates must write artifacts only under runs/<run_name>/. Invalid or timed-out evaluations should print primary_metric: nan when the candidate can exit cleanly.

Compute selection

The generated repo is compute-neutral. It records the selected compute mode in campaign.json and leaves task-specific ML stacks to each task pack's requirements.txt. 

scripts/bootstrap.sh task=<slug> run_tag=<run_tag> device=cpu n_slots=1
scripts/bootstrap.sh task=<slug> run_tag=<run_tag> device=gpu gpus=0,1

Launch and resume

After bootstrap, use the generated launch script. With no agent command, it runs local codex exec automatically as a persistent Bash round loop. Each round writes a fresh prompt, exports AUTORESEARCH_ROUND, runs one agent process, reconciles slots, sleeps, and repeats until the user stops it or repeated command failures hit max_failures. 

scripts/autoresearch_launch.sh task=<slug> run_tag=<run_tag>

Use max_rounds=1 for one bounded agent round. 

scripts/autoresearch_launch.sh task=<slug> run_tag=<run_tag> max_rounds=1

max_turns=1 remains accepted as a compatibility alias.

Set AUTORESEARCH_AGENT_CMD or pass an explicit CLI command after -- to use a different agent. Use agent=prompt to print the persistent orchestration prompt without running an agent. 

scripts/autoresearch_launch.sh task=<slug> run_tag=<run_tag> -- <agent command...>

Interactive report

Lifecycle scripts automatically append structured JSONL events and refresh a self-contained HTML report with clickable idea nodes, green rings for campaign-best improvements, parent-inspiration curves, and a campaign-best metric curve. 

runs/<slug>/<run_tag>/campaign_events.jsonl
runs/<slug>/<run_tag>/report.html

Rebuild the report manually or add a note event: 

python3 scripts/campaign_log.py render --task <slug> --run-tag <run_tag>
python3 scripts/campaign_log.py log --task <slug> --run-tag <run_tag> --event note --message "<note>"

Adapters

The template is agent-agnostic by default. Adapters are optional overlays that add product-specific prompts and guardrails.

Adapter Contents When to use
claude .claude/agents/, .claude/settings.json, and a guard hook. Use when the generated repo will run Claude Code subagents and hooks.
none No adapter files. Use for Codex, another coding agent, or a custom orchestration layer.

Future adapters can be added under skills/autoresearch/assets/adapters/<name>/ and exposed through the scaffolder.

Validation And Development Checks

The generated repository contains its own task-pack validator. Use it after the skill creates or edits a task pack. 

scripts/validate_task.sh tasks/<slug>

Useful checks while developing the skill package: 

python3 -B -c 'import pathlib; [compile(pathlib.Path(p).read_text(), p, "exec") for p in ["skills/autoresearch/scripts/scaffold_repo.py", "skills/autoresearch/assets/repo-template/prepare.py", "skills/autoresearch/assets/repo-template/evaluate.py", "skills/autoresearch/assets/repo-template/scripts/campaign_log.py"]]'
bash -n skills/autoresearch/assets/repo-template/scripts/*.sh
tmp="$(mktemp -d)" && mkdir "$tmp/repo" "$tmp/repo/.omx" && python skills/autoresearch/scripts/scaffold_repo.py --target "$tmp/repo" --name Smoke --no-git
ruby -ryaml -e 'front = File.read("skills/autoresearch/SKILL.md").match(/\A---\n(.*?)\n---/m)[1]; YAML.safe_load(front); puts "skill yaml ok"'
A complete smoke check is to scaffold one default repo and one repo with --adapter claude --include-csp-example, then confirm both are clean git repos on agent/root with the Lore-format initial commit.
